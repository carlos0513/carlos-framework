## 1. 项目概述

### 1.1 项目背景

本项目旨在将 **SoybeanAdmin** (Vue3 前端管理框架) 与 **Carlos Framework** (Spring Boot 后端框架)
进行深度整合，实现一套完整的企业级开发解决方案。

### 1.2 对接目标

| 目标     | 描述                |
|--------|-------------------|
| 统一响应格式 | 前后端响应数据结构保持一致     |
| 完整认证流程 | 实现登录、Token刷新、登出功能 |
| 动态权限路由 | 根据用户角色动态加载菜单路由    |
| 用户角色管理 | 完整的RBAC权限管理功能     |

## 3. 关键数据格式对照

### 3.1 响应结构对照

**后端当前格式** (`Result<T>`):

```json
{
  "code": 200,
  "success": true,
  "message": "操作成功",
  "stack": null,
  "data": { ... }
}
```

**前端期望格式** (`App.Service.Response`):

```json
{
  "code": "0000",
  "msg": "操作成功",
  "data": { ... }
}
```

### 3.2 字段映射表

| 后端字段      | 前端字段   | 后端类型    | 前端类型   | 处理方式         |
|-----------|--------|---------|--------|--------------|
| `code`    | `code` | Integer | String | 需要转换         |
| `success` | -      | Boolean | -      | 前端使用 code 判断 |
| `message` | `msg`  | String  | String | 字段映射         |
| `data`    | `data` | T       | T      | ✅ 一致         |
| `stack`   | -      | String  | -      | 仅错误时返回       |

### 3.3 响应码映射表

| 后端 Code | 前端 Code | 场景       | 处理动作      |
|---------|---------|----------|-----------|
| 200     | "0000"  | 成功       | 正常处理      |
| 401     | "8888"  | 未认证      | 跳转登录页     |
| 403     | "8889"  | 无权限      | 显示无权限页面   |
| 410     | "9999"  | Token 过期 | 自动刷新Token |
| 500     | "5000"  | 系统错误     | 显示错误提示    |

### 3.4 分页结构对照

**前端分页参数**:

```typescript
interface PaginatingCommonParams {
  current: number;  // 当前页码
  size: number;     // 每页大小
  total: number;    // 总记录数
}

interface PaginatingQueryRecord<T> extends PaginatingCommonParams {
  records: T[];     // 数据列表
}
```

**后端分页结构** (`PageInfo<T>` 继承 MyBatis-Plus `Page<T>`):

```java
public class PageInfo<T> extends Page<T> {
    private long current;   // 当前页码
    private long size;      // 每页大小
    private long total;     // 总记录数
    private List<T> records; // 数据列表
}
```

**结论**: ✅ 分页结构完全一致，无需转换。

## Phase 1: 响应格式统一与基础配置

### 1.1 目标

统一前后端响应数据结构，确保数据互通无障碍。

### 1.2 实施方案选择

#### 方案 A: 后端适配前端 (推荐)

**优势**:

- 前端无需改动，降低风险
- 符合 SoybeanAdmin 设计规范
- 后续升级前端框架更方便

**实施步骤**:

1. **修改 Result 类**，添加适配字段

```java
// carlos-spring-boot-core/src/main/java/com/carlos/core/response/Result.java

@Data
@Accessors(chain = true)
@Builder
@AllArgsConstructor
public class Result<T> implements Serializable {

    private Integer code;
    private boolean success;
    private String message;
    private String stack;
    private T data;

    // ========== 新增适配字段 ==========
    
    /**
     * 适配前端 msg 字段
     */
    @JsonProperty("msg")
    public String getMsg() {
        return this.message;
    }
    
    /**
     * 适配前端 code 字段 (String 类型)
     */
    @JsonProperty("code")
    public String getCodeString() {
        return convertCode(this.code);
    }
    
    /**
     * 后端内部使用的 Integer code
     */
    @JsonIgnore
    public Integer getCode() {
        return this.code;
    }
    
    /**
     * 状态码转换规则
     */
    private static String convertCode(Integer code) {
        if (code == null) return "5000";
        return switch (code) {
            case 200 -> "0000";      // 成功
            case 401 -> "8888";      // 未认证
            case 403 -> "8889";      // 无权限
            case 410 -> "9999";      // Token过期
            case 500 -> "5000";      // 系统错误
            default -> String.valueOf(code);
        };
    }
}
```

2. **创建全局响应包装器** (可选)

```java
@ControllerAdvice
public class GlobalResponseAdvice implements ResponseBodyAdvice<Object> {

    @Override
    public boolean supports(MethodParameter returnType, Class<? extends HttpMessageConverter<?>> converterType) {
        // 只处理返回 Result 类型的方法
        return !returnType.getParameterType().equals(Result.class);
    }

    @Override
    public Object beforeBodyWrite(Object body, MethodParameter returnType, 
            MediaType selectedContentType,
            Class<? extends HttpMessageConverter<?>> selectedConverterType,
            ServerHttpRequest request, ServerHttpResponse response) {
        // 自动包装非 Result 类型的返回值
        return Result.success(body);
    }
}
```

#### 方案 B: 前端适配后端

**适用场景**: 后端改动成本过高，或已有多个前端在使用后端接口。

修改 `carlos-framework-admin/src/service/request/index.ts`:

```java
// carlos-auth-service/src/main/java/com/carlos/auth/web/AuthController.java

@RestController
@RequestMapping("/auth")
@RequiredArgsConstructor
@Tag(name = "认证管理")
public class AuthController {

    private final LoginService loginService;
    private final TokenService tokenService;

    @PostMapping("/login")
    @Operation(summary = "用户登录")
    public Result<LoginResponse> login(@RequestBody @Valid LoginRequest request) {
        // 调用现有登录逻辑
        LoginResponse response = loginService.login(request.getUserName(), request.getPassword());
        return Result.success(response);
    }

    @GetMapping("/getUserInfo")
    @Operation(summary = "获取用户信息")
    public Result<UserInfoResponse> getUserInfo(
            @RequestHeader("Authorization") String authorization) {
        // 解析 Token
        String token = authorization.replace("Bearer ", "");
        Long userId = tokenService.getUserIdFromToken(token);
        
        UserInfoResponse userInfo = loginService.getUserInfo(userId);
        return Result.success(userInfo);
    }

    @PostMapping("/refreshToken")
    @Operation(summary = "刷新Token")
    public Result<LoginResponse> refreshToken(@RequestBody RefreshTokenRequest request) {
        LoginResponse response = tokenService.refreshToken(request.getRefreshToken());
        return Result.success(response);
    }

    @PostMapping("/logout")
    @Operation(summary = "用户登出")
    public Result<Void> logout(@RequestHeader("Authorization") String authorization) {
        String token = authorization.replace("Bearer ", "");
        tokenService.invalidateToken(token);
        return Result.success();
    }
}
```

### 2.4 Token 刷新机制

前端已实现自动刷新逻辑，后端需保证:

1. **Token 有效期**: access_token 建议 2 小时，refresh_token 建议 7 天
2. **刷新接口**: 使用 refresh_token 换取新的 access_token
3. **并发控制**: 多个请求同时触发刷新时，只执行一次刷新

```java
@Service
@RequiredArgsConstructor
public class TokenService {
    
    private final RedisTemplate<String, String> redisTemplate;
    
    public LoginResponse refreshToken(String refreshToken) {
        // 1. 验证 refresh_token 有效性
        if (!isValidRefreshToken(refreshToken)) {
            throw new ServiceException("刷新令牌无效或已过期");
        }
        
        // 2. 获取用户ID
        Long userId = getUserIdFromRefreshToken(refreshToken);
        
        // 3. 使旧 token 失效
        invalidateToken(refreshToken);
        
        // 4. 生成新 token
        String newAccessToken = generateAccessToken(userId);
        String newRefreshToken = generateRefreshToken(userId);
        
        return LoginResponse.builder()
            .token(newAccessToken)
            .refreshToken(newRefreshToken)
            .expiresIn(7200L)
            .build();
    }
}
```

```typescript
namespace Route {

    id: string;

    multiTab?: boolean;     // 是否支持多标签
    fixedIndexInTab?: number;
    query?: Record<string, string>; // 默认参数
```

```java
    @GetMapping("/getConstantRoutes")
    @Operation(summary = "获取常量路由")
    public Result<List<MenuRouteVO>> getConstantRoutes() {
        // 获取常量路由（登录页、404等）
        List<Menu> constantMenus = menuService.getConstantMenus();
        List<MenuRouteVO> routes = convertToRouteVO(constantMenus);
        return Result.success(routes);
    }

    @GetMapping("/getUserRoutes")
    @Operation(summary = "获取用户路由")
    public Result<UserRouteVO> getUserRoutes(
            @RequestHeader("Authorization") String authorization) {
        String token = authorization.replace("Bearer ", "");
        Long userId = tokenService.getUserIdFromToken(token);
        
        // 获取用户角色
        List<String> roles = getUserRoles(userId);
        
        // 根据角色获取菜单
        List<Menu> menus = menuService.getMenusByRoles(roles);
        List<MenuRouteVO> routes = convertToRouteVO(menus);
        
        // 确定首页
        String homeRoute = determineHomeRoute(routes);
        
        UserRouteVO result = new UserRouteVO();
        result.setRoutes(routes);
        result.setHome(homeRoute);
        
        return Result.success(result);
    }

    @GetMapping("/isRouteExist")
    @Operation(summary = "判断路由是否存在")
    public Result<Boolean> isRouteExist(@RequestParam String routeName) {
        boolean exists = menuService.isRouteExists(routeName);
        return Result.success(exists);
    }

    // ========== 转换方法 ==========
    
    private List<MenuRouteVO> convertToRouteVO(List<Menu> menus) {
        return menus.stream()
            .map(this::convertSingleMenu)
            .collect(Collectors.toList());
    }
    
    private MenuRouteVO convertSingleMenu(Menu menu) {
        MenuRouteVO vo = new MenuRouteVO();
        vo.setId(String.valueOf(menu.getId()));
        vo.setName(menu.getRouteName());
        vo.setPath(menu.getRoutePath());
        vo.setComponent(menu.getComponent());
        
        RouteMetaVO meta = new RouteMetaVO();
        meta.setTitle(menu.getMenuName());
        meta.setIcon(menu.getIcon());
        meta.setI18nKey(menu.getI18nKey());
        meta.setKeepAlive(menu.getKeepAlive());
        meta.setConstant(menu.getConstant());
        meta.setOrder(menu.getSortOrder());
        meta.setHideInMenu(menu.getHideInMenu());
        meta.setActiveMenu(menu.getActiveMenu());
        
        // 解析 query 参数
        if (StrUtil.isNotBlank(menu.getQuery())) {
            meta.setQuery(JSONUtil.toBean(menu.getQuery(), Map.class));
        }
        
        vo.setMeta(meta);
        
        // 递归处理子菜单
        if (CollUtil.isNotEmpty(menu.getChildren())) {
            vo.setChildren(convertToRouteVO(menu.getChildren()));
        }
        
        return vo;
    }
    
    private String determineHomeRoute(List<MenuRouteVO> routes) {
        // 查找第一个非隐藏的路由作为首页
        return routes.stream()
            .filter(r -> !Boolean.TRUE.equals(r.getMeta().getHideInMenu()))
            .findFirst()
            .map(MenuRouteVO::getName)
            .orElse("home");
    }
}
```

## 风险评估与应对

### 风险 1: 响应格式改动影响现有接口

**风险等级**: 高  
**影响范围**: 所有已有接口  
**应对措施**:

- 采用向后兼容的方式修改 Result 类
- 添加新的适配字段，保留原有字段
- 测试所有现有接口

### 风险 2: Token 刷新并发问题

**风险等级**: 中  
**影响范围**: 用户体验  
**应对措施**:

- 前端已实现请求队列，确保刷新时其他请求等待
- 后端确保 refresh_token 只能使用一次

### 风险 3: 菜单数据与前端路由不匹配

**风险等级**: 中  
**影响范围**: 页面访问  
**应对措施**:

- 建立菜单-路由映射规范文档
- 组件路径统一命名规范
- 添加路由存在性检查接口

### 风险 4: 权限控制粒度不一致

**风险等级**: 中  
**影响范围**: 系统安全  
**应对措施**:

- 统一按钮权限编码规范：`{模块}:{操作}`，如 `user:add`
- 前后端权限校验保持一致
- 定期审计权限配置

```java
@Configuration
@EnableKnife4j
public class Knife4jConfig {
    @Bean
    public OpenAPI customOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("Carlos Admin API")
                .version("1.0")
                .description("Carlos 管理后台 API 文档"));
    }
}
```

访问地址: `http://localhost:8080/doc.html`

### B. 前端类型生成工具

可以使用工具根据后端 Swagger 生成前端类型:

```bash
# 使用 swagger-typescript-api
npx swagger-typescript-api -p http://localhost:8080/v3/api-docs -o ./src/typings -n api-generated.d.ts
```

## 二、严重性能问题（🔴 高优先级）

### 2.1 WafFilter - Web应用防火墙

**文件**: `security/WafFilter.java`

#### 问题列表

| # | 问题               | 影响                             | 优化建议                                 |
|---|------------------|--------------------------------|--------------------------------------|
| 1 | **Pattern 重复编译** | 每次请求都创建 Matcher                | 使用 `ThreadLocal<Matcher>` 或预编译缓存     |
| 2 | **请求体重复读取**      | `DataBufferUtils.join()` 阻塞式读取 | 使用 `ServerHttpRequestDecorator` 包装缓存 |
| 3 | **正则表达式回溯**      | 复杂正则可能导致 ReDoS                 | 使用确定性自动机 (DFA) 替代 NFA                |
| 4 | **Header 全量遍历**  | O(N) 遍历所有 Header               | 建立关键 Header 索引                       |

#### 代码问题示例

```java
// ❌ 问题：每次请求都创建 Matcher
Matcher sqlMatcher = SQL_INJECTION_PATTERN.matcher(query);
if (sqlMatcher.find()) { ... }

// ❌ 问题：每次请求都遍历所有 Header
for (String headerName : headers.keySet()) {
    List<String> values = headers.get(headerName);
    ...
}
```

#### 优化方案

```java
// ✅ 使用 ThreadLocal 复用 Matcher
private static final ThreadLocal<Matcher> SQL_MATCHER = 
    ThreadLocal.withInitial(() -> SQL_INJECTION_PATTERN.matcher(""));

// ✅ 分层检测策略
private boolean fastCheck(String input) {
    // 1. 快速字符检查（无正则）
    if (!containsDangerousChars(input)) return false;
    // 2. 精确匹配（HashSet）
    if (EXACT_PATTERNS.contains(input)) return true;
    // 3. 正则匹配（最后手段）
    return regexCheck(input);
}
```

**预期收益**: QPS 提升 3-5x，延迟降低 50-70%

**文件**: `oauth2/OAuth2AuthorizationFilter.java`

#### 问题列表

| # | 问题              | 影响                                        | 优化建议                       |
|---|-----------------|-------------------------------------------|----------------------------|
| 1 | **权限匹配使用正则**    | `matchPermission()` 使用 `String.matches()` | 改用 `PathMatchUtil` 或缓存匹配结果 |
| 2 | **无权限缓存**       | 每次请求都从 Redis 获取权限                         | 使用 Caffeine 本地缓存           |
| 3 | **ABAC 属性重复构建** | 每次请求都创建新的 Map                             | 使用对象池或不可变对象                |

#### 代码问题

```java
// ❌ 问题：每次请求都编译正则
if (required.matches(permission.replace("**", ".*"))) {
    return true;
}

// ❌ 问题：频繁创建 Map
Map<String, Object> attributes = Map.of(
    "userId", userContext.getUserId(),
    "roleIds", userContext.getRoleIds(),
    ...
);
```

**预期收益**: 鉴权延迟从 5-10ms 降低到 0.5-1ms

javax.crypto.Mac mac = javax.crypto.Mac.getInstance("HmacSHA256");
mac.init(new SecretKeySpec(key.getBytes(), "HmacSHA256"));

// ✅ 优化：使用 ThreadLocal
private static final ThreadLocal<Mac> HMAC_SHA256 = ThreadLocal.withInitial(() -> {

    }
});

### 3.5 MetricsFilter - 指标收集

**文件**: `observability/MetricsFilter.java`

#### 问题

| # | 问题              | 优化建议               |
|---|-----------------|--------------------|
| 1 | 每次请求都注册 Counter | 使用静态缓存的 Counter 实例 |
| 2 | 高基数标签（path）     | 使用路径模式替代完整路径       |

#### 优化代码

```java
// ❌ 问题：每次请求都注册
Counter.builder(REQUEST_COUNTER)
    .tag("route", routeId)
    .tag("path", path)  // 高基数！
    .register(meterRegistry)
    .increment();

// ✅ 优化：使用静态缓存 + 路径模式
private final ConcurrentHashMap<String, Counter> counterCache = new ConcurrentHashMap<>();

private Counter getCounter(String routeId, String pathPattern) {
    return counterCache.computeIfAbsent(routeId + ":" + pathPattern,
        k -> Counter.builder(REQUEST_COUNTER)
            .tag("route", routeId)
            .tag("path_pattern", pathPattern)
            .register(meterRegistry));
}
```

## 五、整体优化建议

### 5.1 过滤器执行顺序优化

```
当前顺序：
RequestTracingFilter (-2950) → WafFilter (-5000) → ReplayProtectionFilter (-4000)

建议顺序：
RequestTracingFilter → WafFilter → ReplayProtectionFilter → PathPrefixFilter
                        ↓
                   [尽早拦截攻击流量]
```

### 5.2 缓存策略汇总

| 缓存类型        | 适用过滤器                                | 建议配置                       |
|-------------|--------------------------------------|----------------------------|
| Caffeine    | WafFilter, OAuth2AuthorizationFilter | maxSize=1000, expire=10min |
| ThreadLocal | HMAC计算, Pattern匹配                    | 无过期，线程绑定                   |
| Redis       | 黑名单, 权限数据                            | 5min过期                     |

### 5.3 性能监控指标

建议添加以下监控：

```java
// 过滤器执行时间
Timer.builder("gateway.filter.duration")
    .tag("filter", filterName)
    .register(meterRegistry);

// 缓存命中率
Gauge.builder("gateway.filter.cache.hit_ratio")
    .valueFn(() -> cache.stats().hitRate())
    .register(meterRegistry);
```

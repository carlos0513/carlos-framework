## 2. 核心功能清单

### 2.1 认证模块 [AUTH]

#### AUTH-001 用户名密码登录

- **功能描述**: 支持用户名/邮箱/手机号 + 密码登录
- **输入**: username/email/phone (三选一) + password
- **处理逻辑**:
    1. 根据输入类型查询用户 (username/email/phone)
    2. BCrypt验证密码 (strength=10)
    3. 检查账号状态 (启用/禁用/锁定)
    4. 记录登录审计日志
    5. 生成Session或JWT (根据配置)
- **失败处理**:
    - 连续5次失败锁定账号15分钟
    - 第3次失败起要求输入验证码
    - 返回模糊错误: "用户名或密码错误"
- **输出**:
    - 成功: Session Cookie 或 JWT Token
    - 失败: 错误码 + 剩余尝试次数

#### AUTH-002 验证码登录

- **功能描述**: 短信/邮箱验证码登录
- **验证码规则**:
    - 格式: 6位数字
    - 有效期: 5分钟 (Redis TTL)
    - 存储: Redis Key = `auth:captcha:{type}:{target}`
- **防刷限制**:
    - 发送间隔: 60秒
    - 日上限: 同一手机号/邮箱 10条/天
    - 计数Key: `auth:captcha:limit:{type}:{target}`
- **流程**:
    1. 用户输入手机号/邮箱
    2. 后端生成验证码，存入Redis
    3. 调用短信/邮件服务发送，采用预留接口方式，具体发送实现自行扩展
    4. 用户输入验证码验证
    5. 验证通过则登录成功 (无密码用户自动注册)

#### AUTH-003 第三方OAuth登录

- **支持渠道**: 微信(work.weixin.qq.com)、钉钉、飞书、GitHub、Google、其他企业系统（如何灵活扩展请做好接口设计）
- **流程**:
    1. 前端点击图标，跳转 `/login/{provider}`
    2. 后端生成state，存入Session，重定向到授权URL
    3. 用户在第三方确认授权
    4. 回调到 `/login/{provider}/callback`
    5. 用code换取access_token，获取用户信息
    6. 根据unionid/openid查询绑定关系
    7. 已绑定: 直接登录 | 未绑定: 引导注册或绑定已有账号
- **绑定机制**:
    - 登录状态下绑定: 关联到当前用户
    - 未登录状态下: 自动创建新账号 (需补充手机号)

#### AUTH-004 多因素认证(MFA)

- **TOTP实现**: RFC 6238，兼容Google Authenticator
- **启用流程**:
    1. 用户开启MFA，后端生成随机secret
    2. 返回QR Code (otpauth:// URI)
    3. 用户用App扫码，输入6位验证码验证
    4. 验证通过后启用，生成10组备用恢复码
- **验证触发场景**:
    - 异地登录 (IP地理位置变化)
    - 新设备登录 (User-Agent/设备指纹变化)
    - 敏感操作 (修改密码、绑定手机等)
- **备用恢复码**: 10组8位随机码，使用后失效，可申请重置

#### AUTH-005 扫码登录

- **流程**:
    1. PC端请求生成二维码，后端生成唯一qr_id，返回二维码图片
    2. Redis存储: `auth:qr:{qr_id}` = {status: PENDING, expire: now+5min}
    3. PC端轮询状态 (每2秒)
    4. App扫描二维码，解析qr_id，发送确认请求
    5. 状态变为 SCANNED，PC端展示确认界面
    6. 用户在App点击确认，携带登录凭证
    7. 后端验证App凭证，状态变为 CONFIRMED，存储user_id
    8. PC端轮询到CONFIRMED，获取登录态
- **安全**: 二维码绑定设备指纹，防止截图盗用

### 2.2 OAuth2.1/OIDC模块 [OAUTH]

#### OAUTH-001 授权码模式 (Authorization Code + PKCE)

- **端点**: `GET /oauth2/authorize`
- **必需参数**:
    - `response_type=code`
    - `client_id`: 注册的应用ID
    - `redirect_uri`: 必须在注册列表中
    - `scope`: 申请权限范围，空格分隔
    - `state`: CSRF防护随机串 (推荐)
    - `code_challenge`: PKCE码 (S256方法)
    - `code_challenge_method=S256`
- **流程**:
    1. 验证client_id和redirect_uri
    2. 检查用户登录态，未登录跳转登录页
    3. 展示授权确认页 (scope列表)
    4. 用户确认后生成授权码 (32位随机串)
    5. 授权码存Redis，TTL=5分钟，一次性使用
    6. 重定向到redirect_uri?code=xxx&state=xxx
- **PKCE验证**: Token端点验证code_verifier与code_challenge

#### OAUTH-002 令牌颁发

- **端点**: `POST /oauth2/token`
- **支持模式**:
    - `authorization_code`: 授权码换令牌
    - `refresh_token`: 刷新令牌
    - `client_credentials`: 客户端凭证 (服务间调用)
- **认证方式**:
    - `client_secret_basic`: Authorization Header Base64(client_id:secret)
    - `client_secret_post`: 请求体中client_id和client_secret
    - `private_key_jwt`: JWT客户端断言 (高安全场景)
- **令牌规格**:
    - Access Token: JWT, RS256签名, 15分钟有效期
    - Refresh Token: 随机字符串, Redis存储, 7天有效期
    - ID Token: OIDC标准JWT, 包含用户claims
- **响应示例**:
  ```json
  {
    "access_token": "eyJraWQiOiIx...",
    "token_type": "Bearer",
    "expires_in": 900,
    "refresh_token": "eyJraWQiOiIy...",
    "scope": "openid profile email",
    "id_token": "eyJraWQiOiIz..."
  }
  ```

#### OAUTH-003 令牌管理

- **刷新机制**:
    - 请求: `grant_type=refresh_token` + `refresh_token=xxx`
    - 验证refresh_token有效性 (Redis存在且未过期)
    - 生成新access_token，可选生成新refresh_token (轮换)
    - 旧refresh_token立即失效 (防重放)
- **吊销机制**:
    - 端点: `POST /oauth2/revoke`
    - 参数: `token=xxx` + `token_type_hint=access_token/refresh_token`
    - 实现: 将token jti存入Redis黑名单，TTL=token剩余有效期
- **自省机制**:
    - 端点: `POST /oauth2/introspect` (RFC 7662)
    - 供资源服务器验证令牌，返回active状态及claims

#### OAUTH-004 客户端管理

基于现有com.carlos.oauth.app目录下的相关实现

### 2.3 会话管理 [SESSION]

#### SESSION-001 单点登录(SSO)

- **方案选择** (配置切换):
    - **Session方案**: 基于Redis的分布式Session，Cookie域共享
    - **JWT方案**: 无状态，Access Token存储在Cookie或LocalStorage
- **Session方案细节**:
    - Session ID: 32字节随机字符串
    - Cookie属性: HttpOnly, Secure, SameSite=Lax, Path=/
    - Redis Key: `auth:session:{sessionId}` -> UserDetails JSON
    - TTL: 30分钟空闲过期，每次访问刷新
- **登录态传播**:
    - 子系统检测到未登录，重定向到认证中心
    - 认证中心检查Session，已登录则签发令牌重定向回去
    - 子系统用令牌换取本地Session

#### SESSION-002 单点登出(SLO)

- **Back-Channel Logout** (推荐):
    - 用户点击登出，认证中心清除Session
    - 查询该用户所有已登录客户端
    - 服务端并发调用各客户端logout回调URL
    - 客户端清除本地Session，返回确认
- **Front-Channel Logout** (兼容旧系统):
    - 登出后渲染页面，内含多个iframe
    - 每个iframe指向客户端登出端点
    - 浏览器并行请求，清除各域Cookie
- **OIDC RP-Initiated Logout**:
    - 端点: `/connect/logout`
    - 参数: `id_token_hint`, `post_logout_redirect_uri`, `state`
    - 验证后执行全局登出

#### SESSION-003 并发控制

- **策略配置** (`auth.server.session.concurrent-strategy`):
    - `ALLOW`: 允许多端同时在线 (默认)
    - `DENY`: 新登录踢掉旧会话
    - `LIMIT_N`: 最多N个会话，新登录踢掉最老的
- **实现**:
    - Redis ZSet: `auth:user:sessions:{userId}` -> {score: timestamp, member: sessionId}
    - 登录时检查策略，超出限制则删除最老的session
    - 被踢掉的会话下次访问时返回"已在其他地方登录"

### 2.4 用户、组织架构、角色管理 [USER]

同过预留接口，调用用户服务实现

### 2.5 审计日志 [AUDIT]

#### AUDIT-001 登录审计

- **记录字段**: id, user_id, client_id, event_type (LOGIN/LOGOUT/REFRESH), ip, user_agent, location, status,
  error_message, created_at
- **存储策略**:
    - 异步写入 (Spring Event + 线程池)
    - 按月分表 (partition by range)
    - 保留90天，过期归档到对象存储

#### AUDIT-002 操作审计

- **敏感操作列表**:
    - 密码修改/重置
    - MFA绑定/解绑
    - 授权许可 (用户同意第三方应用授权)
    - 权限变更 (角色分配)
    - 账号注销
- **记录内容**: 操作人、操作对象、变更前、变更后、操作时间

#### AUDIT-003 安全告警

- **告警规则**:
  | 规则 | 条件 | 动作 |
  |------|------|------|
  | 异地登录 | IP归属地与常用地不同 | 邮件+短信通知 |
  | 非常用设备 | User-Agent变化 | 要求MFA验证 |
  | 暴力破解 | 5分钟20次登录失败 | IP封禁1小时+告警 |
  | 非工作时间 | 22:00-06:00登录 | 记录并通知管理员 |
- **通知渠道**: 邮件、短信、钉钉Webhook

## 3. 数据库设计

### 3.1 核心表结构

```sql
-- OAuth2授权记录表
CREATE TABLE oauth2_authorization_record (
    id VARCHAR(100) PRIMARY KEY,
    registered_client_id VARCHAR(100) NOT NULL,
    principal_name VARCHAR(200) NOT NULL,
    authorization_grant_type VARCHAR(100) NOT NULL,
    authorized_scopes VARCHAR(1000),
    attributes TEXT COMMENT 'OAuth2Authorization序列化',
    state VARCHAR(500),
    authorization_code_value TEXT,
    authorization_code_issued_at DATETIME,
    authorization_code_expires_at DATETIME,
    access_token_value TEXT,
    access_token_issued_at DATETIME,
    access_token_expires_at DATETIME,
    refresh_token_value TEXT,
    refresh_token_issued_at DATETIME,
    oidc_id_token_value TEXT,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_auth_principal ON oauth2_authorization(principal_name);
CREATE INDEX idx_auth_access_token ON oauth2_authorization((access_token_value(255)));

-- 登录审计表 (按月分区)
CREATE TABLE oauth2_audit_login (
    id BIGSERIAL,
    user_id BIGINT,
    client_id VARCHAR(100),
    event_type VARCHAR(50) NOT NULL,
    ip_address INET NOT NULL,
    user_agent VARCHAR(500),
    location VARCHAR(200),
    status VARCHAR(20) NOT NULL,
    error_message VARCHAR(500),
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id, created_at)
) PARTITION BY RANGE (created_at);

-- 创建分区表示例
CREATE TABLE audit_login_2024_01 PARTITION OF audit_login
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

```

## 5. 安全规范

### 5.1 传输安全

- **TLS**: 强制TLS 1.2+，禁用SSLv3/TLS 1.0/1.1
- **HSTS**: `Strict-Transport-Security: max-age=31536000; includeSubDomains`
- **证书固定**: 可选实现HPKE或证书固定

### 5.2 认证安全

- **密码存储**: BCrypt (cost=10)，禁止MD5/SHA1/明文
- **JWT签名**: RS256非对称算法，密钥长度2048bit+
- **密钥管理**:
    - 私钥存储在JKS或HSM中
    - 支持密钥轮换，新旧密钥同时生效
    - JWKS端点暴露公钥，kid标识密钥版本
- **Session安全**:
    - Session ID: 加密安全的随机数 (SecureRandom)
    - Cookie属性: HttpOnly, Secure, SameSite=Lax
    - 定期轮换Session ID (登录后)

### 5.3 防护机制

| 威胁    | 防护措施             | 实现方式                           |
|-------|------------------|--------------------------------|
| 暴力破解  | 速率限制+账号锁定        | Redis计数，渐进式延迟                  |
| 重放攻击  | 时间戳+Nonce        | 请求时间±5分钟有效，Redis去重             |
| CSRF  | State参数+SameSite | OAuth2 state校验，Cookie SameSite |
| SQL注入 | 参数化查询            | JPA/Hibernate预编译语句             |
| XSS   | 输入过滤+输出编码        | 正则过滤，HTML转义，CSP                |
| 点击劫持  | X-Frame-Options  | DENY或SAMEORIGIN                |
| 信息泄露  | 错误脱敏             | 生产环境不返回堆栈                      |

### 5.4 隐私保护

- **字段加密**: 手机号、邮箱AES-256-GCM存储，密钥分离管理
- **日志脱敏**: 密码、Token、身份证号等字段掩码 (如138****8888)
- **数据最小化**: 仅收集必要信息，明确告知用途
- **GDPR支持**:
    - 数据导出: 用户可导出个人数据
    - 被遗忘权: 删除账号及关联数据 (软删除+定期清理)
    - 同意管理: 记录用户授权同意时间、范围

## 6. 配置规范

### 6.1 应用配置 (application.yml)

```yaml

spring:
  application:
    name: auth-server


auth:
  server:
    issuer: ${AUTH_ISSUER:https://auth.company.com}

    token:
      access-token-ttl: 900        # 15分钟
      refresh-token-ttl: 604800    # 7天
      refresh-token-rotation: true # 轮换机制
      id-token-ttl: 900

    session:
      timeout: 1800                # 30分钟
      concurrent-strategy: DENY    # ALLOW/DENY/LIMIT_N
      max-sessions: 3              # LIMIT_N时生效
      cookie-name: AUTH_SESSION
      cookie-domain: .company.com

    password:
      bcrypt-strength: 10
      min-length: 8
      max-length: 64
      require-complexity: true     # 大小写+数字+特殊字符
      history-count: 5             # 不能与前5次重复
      max-age: 90                  # 天，0表示不限制

    security:
      login-fail-threshold: 5      # 失败锁定阈值
      lock-duration: 900           # 锁定15分钟
      captcha-threshold: 3         # 第3次起需验证码
      captcha-length: 6
      captcha-ttl: 300             # 5分钟
      mfa-enabled: true
      mfa-issuer-name: CompanyAuth

    cors:
      allowed-origins: ["https://*.company.com", "https://localhost:3000"]
      allowed-methods: ["GET", "POST", "PUT", "DELETE", "OPTIONS"]
      allowed-headers: ["*"]
      allow-credentials: true
      max-age: 3600

    rate-limit:
      login:
        capacity: 5
        refill-rate: 1
        refill-period: 10s         # 每10秒恢复1次
      api:
        capacity: 100
        refill-rate: 10
        refill-period: 1s

  jwt:
    key-store: classpath:auth.jks
    key-store-password: ${JKS_PASSWORD:}
    key-alias: auth-key
    key-password: ${KEY_PASSWORD:}

```

## 8. 对接指南

### 8.1 业务系统对接步骤

1. **注册应用**
    - 访问管理后台 `/admin/clients`
    - 填写应用名称、回调地址、授权类型
    - 获取 `client_id` 和 `client_secret`

2. **配置OAuth2参数**
   ```yaml
   spring:
     security:
       oauth2:
         client:
           registration:
             auth-center:
               client-id: your-client-id
               client-secret: your-secret
               scope: openid,profile,email
               authorization-grant-type: authorization_code
               redirect-uri: "{baseUrl}/login/oauth2/code/auth-center"
           provider:
             auth-center:
               issuer-uri: https://auth.company.com
   ```

3. **集成登录按钮**
    - 跳转链接:
      `https://auth.company.com/oauth2/authorize?response_type=code&client_id=xxx&redirect_uri=xxx&scope=openid&state=xxx&code_challenge=xxx&code_challenge_method=S256`
    - 或使用Spring Security默认配置

4. **处理回调**
    - 接收 `code` 和 `state`
    - 校验state防CSRF
    - 用code换token (POST /oauth2/token)

5. **验证令牌**
    - 本地验签: 从JWKS端点获取公钥，验证JWT签名
    - 或远程自省: POST /oauth2/introspect

6. **实现登出**
    - 本地清除Session
    - 重定向到 `https://auth.company.com/connect/logout?id_token_hint=xxx&post_logout_redirect_uri=xxx`

### 8.2 常见问题排查

| 问题                      | 原因                 | 解决                   |
|-------------------------|--------------------|----------------------|
| `invalid_client`        | client_id或secret错误 | 检查配置，确认无空格           |
| `invalid_grant`         | 授权码过期或重复使用         | 授权码5分钟有效，一次性使用       |
| `redirect_uri_mismatch` | 回调地址不匹配            | 必须与注册时完全一致，包括端口和路径   |
| `invalid_scope`         | 请求的scope未授权        | 在管理后台申请scope，用户授权时勾选 |
| `access_denied`         | 用户拒绝授权             | 正常流程，引导用户重新授权        |
| 验签失败                    | 密钥轮换或算法不匹配         | 同步JWKS，检查kid和alg     |
| CORS错误                  | 跨域配置问题             | 联系管理员添加origin到白名单    |

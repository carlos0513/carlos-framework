# 用户中心模块需求文档

> 文档版本: v1.0.0  
> 创建日期: 2026-03-02  
> 所属项目: Carlos Framework  
> 模块名称: carlos-org 用户中心

---

## 1. 文档概述

### 1.1 目的

本文档详细定义 Carlos Framework 用户中心模块的功能需求、业务规则、接口规范及非功能性需求，为开发团队提供清晰的技术实现指导。

### 1.2 适用范围

本文档适用于用户中心模块（carlos-org）的设计、开发、测试及维护人员。

### 1.3 术语定义

| 术语   | 定义                                 |
|------|------------------------------------|
| 租户   | 系统多租户隔离的基本单位，不同租户数据完全隔离            |
| 职系   | 岗位类别，如管理系(M)、技术系(T)、专业系(P)等        |
| 职级   | 同一职系内的等级划分，如 T1、T2-1、M3 等          |
| 主部门  | 用户的主要归属部门，一个用户只有一个主部门              |
| 主岗位  | 用户的主要任职岗位，一个用户可有多个岗位但只有一个主岗位       |
| 数据权限 | 角色可查看的数据范围：全部/本部门及子部门/仅本部门/仅本人/自定义 |

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 用户基本信息管理

**功能描述:**
管理企业员工的账号信息，包括账号创建、编辑、禁用、启用、删除等操作。

**核心字段:**

| 字段名             | 类型           | 必填 | 说明               |
|-----------------|--------------|----|------------------|
| id              | BIGINT       | 是  | 主键，雪花ID          |
| account         | VARCHAR(20)  | 是  | 用户名，租户内唯一        |
| nickname        | VARCHAR(20)  | 是  | 昵称/真实姓名          |
| pwd             | VARCHAR(64)  | 是  | 密码，SM2加密存储       |
| identify        | VARCHAR(18)  | 否  | 证件号码（身份证）        |
| phone           | VARCHAR(11)  | 否  | 手机号              |
| email           | VARCHAR(30)  | 否  | 邮箱               |
| gender          | TINYINT      | 否  | 性别：0保密, 1男, 2女   |
| avatar          | VARCHAR(200) | 否  | 头像URL            |
| state           | TINYINT      | 是  | 状态：0禁用, 1启用, 2锁定 |
| main_dept_id    | BIGINT       | 否  | 主部门ID            |
| login_time      | DATETIME     | 否  | 最后登录时间           |
| login_ip        | VARCHAR(50)  | 否  | 最后登录IP           |
| login_count     | INT          | 是  | 登录次数             |
| pwd_last_modify | DATETIME     | 是  | 密码最后修改时间         |

**业务规则:**

1. **账号唯一性**: 用户名在同一租户内必须唯一
2. **密码安全**:
    - 密码必须包含大小写字母、数字、特殊字符中的至少3种
    - 密码长度不少于8位
    - 使用SM2国密算法加密存储
    - 默认90天必须修改密码
3. **状态流转**:
    - 启用 → 禁用 → 启用
    - 启用 → 锁定（连续5次密码错误）→ 启用（管理员解锁或30分钟后自动解锁）
4. **主部门约束**: 用户必须有且只有一个主部门
5. **逻辑删除**: 删除操作为逻辑删除，保留历史数据

**功能清单:**

| 功能编号   | 功能名称    | 功能描述           | 优先级 |
|--------|---------|----------------|-----|
| UM-001 | 用户列表查询  | 支持分页、条件筛选、模糊搜索 | P0  |
| UM-002 | 用户详情查询  | 查询用户完整信息及关联关系  | P0  |
| UM-003 | 新增用户    | 创建用户账号，支持批量导入  | P0  |
| UM-004 | 编辑用户    | 修改用户基本信息       | P0  |
| UM-005 | 删除用户    | 逻辑删除用户         | P0  |
| UM-006 | 启用/禁用用户 | 切换用户状态         | P0  |
| UM-007 | 解锁用户    | 管理员解锁被锁定的账号    | P1  |
| UM-008 | 重置密码    | 管理员重置用户密码      | P0  |
| UM-009 | 修改密码    | 用户自主修改密码       | P0  |
| UM-010 | 用户导入    | Excel批量导入用户    | P1  |
| UM-011 | 用户导出    | 导出用户列表到Excel   | P1  |
| UM-012 | 用户分配部门  | 设置用户主部门及兼职部门   | P0  |
| UM-013 | 用户分配角色  | 为用户分配角色        | P0  |
| UM-014 | 用户分配岗位  | 设置用户主岗位及兼岗     | P1  |

#### 3.1.2 用户导入功能

**功能描述:**
通过Excel批量导入用户数据，支持模板下载、数据校验、错误反馈。

**导入字段:**

| 字段名      | 必填 | 校验规则               |
|----------|----|--------------------|
| account  | 是  | 用户名，租户内唯一，4-20字符   |
| realname | 是  | 真实姓名，2-20字符        |
| identify | 否  | 身份证号，18位，校验格式      |
| phone    | 是  | 手机号，11位数字          |
| role     | 否  | 角色名称，需存在           |
| dept     | 否  | 部门路径，如"总部-技术部-研发组" |
| gender   | 否  | 性别：男/女/保密          |
| email    | 否  | 邮箱格式校验             |

**导入流程:**

```
1. 下载导入模板
       │
       ▼
2. 填写数据并上传Excel
       │
       ▼
3. 系统校验数据格式
   ├─ 格式错误 → 返回错误报告
   └─ 校验通过 → 继续
       │
       ▼
4. 数据唯一性校验（账号、手机号）
   ├─ 重复 → 标记冲突，支持覆盖/跳过
   └─ 通过 → 继续
       │
       ▼
5. 关联数据校验（部门、角色）
   ├─ 不存在 → 标记错误
   └─ 存在 → 继续
       │
       ▼
6. 预览导入结果
       │
       ▼
7. 确认导入 → 批量写入数据库
       │
       ▼
8. 发送通知（邮件/短信）给新用户
```

### 3.2 部门管理模块

#### 3.2.1 部门组织架构

**功能描述:**
管理企业组织架构，支持多级部门、部门树展示、部门合并与拆分。

**核心字段:**

| 字段名       | 类型           | 必填 | 说明               |
|-----------|--------------|----|------------------|
| id        | BIGINT       | 是  | 主键               |
| parent_id | BIGINT       | 是  | 父部门ID，0为根部门      |
| dept_name | VARCHAR(64)  | 是  | 部门名称             |
| dept_code | VARCHAR(64)  | 否  | 部门编号，唯一          |
| path      | VARCHAR(500) | 是  | 部门路径，如"/1/5/23/" |
| leader_id | BIGINT       | 否  | 负责人ID            |
| state     | TINYINT      | 是  | 状态：0禁用, 1启用      |
| sort      | TINYINT      | 是  | 同级排序             |
| level     | INT          | 是  | 层级深度             |

**业务规则:**

1. **层级限制**: 部门层级最多支持10级
2. **唯一性**: 部门编号在同一租户内唯一
3. **删除约束**: 部门下有子部门或用户时，不允许删除
4. **path维护**: 部门移动时自动更新所有子部门的path
5. **负责人**: 部门负责人必须是该部门或其子部门的成员

**功能清单:**

| 功能编号   | 功能名称    | 功能描述        | 优先级 |
|--------|---------|-------------|-----|
| DM-001 | 部门树查询   | 查询完整部门树结构   | P0  |
| DM-002 | 部门列表    | 平铺列表展示，支持筛选 | P0  |
| DM-003 | 新增部门    | 创建新部门       | P0  |
| DM-004 | 编辑部门    | 修改部门信息      | P0  |
| DM-005 | 删除部门    | 删除空部门       | P0  |
| DM-006 | 移动部门    | 调整部门层级      | P1  |
| DM-007 | 部门排序    | 调整同级部门顺序    | P1  |
| DM-008 | 部门人员列表  | 查看部门下的用户    | P0  |
| DM-009 | 设置部门负责人 | 指定部门负责人     | P1  |
| DM-010 | 部门角色配置  | 设置部门默认角色    | P2  |

#### 3.2.2 部门编制管理

**功能描述:**
管理各部门的人员编制，包括编制数设置、在编统计、超编预警。

**业务规则:**

1. **编制类型**: 标准岗有编制限制，虚拟岗/兼职岗无编制限制
2. **编制统计**: 实时计算各岗位的在编人数和空缺人数
3. **超编预警**: 在编人数超过编制数时发出预警
4. **编制冻结**: 部门可设置冻结编制，暂停招聘

### 3.3 角色权限管理模块

#### 3.3.1 角色管理

**功能描述:**
管理系统角色，支持系统预设角色和自定义角色，实现RBAC权限模型。

**核心字段:**

| 字段名        | 类型           | 必填 | 说明                                    |
|------------|--------------|----|---------------------------------------|
| id         | BIGINT       | 是  | 主键                                    |
| role_name  | VARCHAR(40)  | 是  | 角色名称                                  |
| role_code  | VARCHAR(100) | 否  | 角色编码，唯一                               |
| role_type  | TINYINT      | 是  | 类型：1系统角色, 2自定义角色                      |
| data_scope | TINYINT      | 是  | 数据范围：1全部, 2本部门及子部门, 3仅本部门, 4仅本人, 5自定义 |
| state      | TINYINT      | 是  | 状态：0禁用, 1启用                           |

**数据权限范围:**

| 类型      | 编码 | 说明             |
|---------|----|----------------|
| 全部数据    | 1  | 可查看所有数据（超级管理员） |
| 本部门及子部门 | 2  | 可查看本部门及所有子部门数据 |
| 仅本部门    | 3  | 仅可查看本部门数据      |
| 仅本人     | 4  | 仅可查看自己创建的数据    |
| 自定义规则   | 5  | 通过自定义SQL规则控制   |

**功能清单:**

| 功能编号   | 功能名称    | 功能描述             | 优先级 |
|--------|---------|------------------|-----|
| RM-001 | 角色列表    | 分页查询角色           | P0  |
| RM-002 | 角色详情    | 查看角色详情及权限        | P0  |
| RM-003 | 新增角色    | 创建自定义角色          | P0  |
| RM-004 | 编辑角色    | 修改角色信息           | P0  |
| RM-005 | 删除角色    | 删除未使用的角色         | P0  |
| RM-006 | 启用/禁用角色 | 切换角色状态           | P1  |
| RM-007 | 配置角色权限  | 为角色分配菜单/按钮/API权限 | P0  |
| RM-008 | 配置数据权限  | 设置角色数据范围         | P0  |
| RM-009 | 查看角色用户  | 查看拥有该角色的用户       | P1  |
| RM-010 | 复制角色    | 基于现有角色创建新角色      | P2  |

### 3.4 岗位管理模块

#### 3.4.1 职系管理

**功能描述:**
管理岗位类别（职系），定义职业发展通道。

**核心字段:**

| 字段名           | 类型           | 必填 | 说明                    |
|---------------|--------------|----|-----------------------|
| id            | BIGINT       | 是  | 主键                    |
| category_code | VARCHAR(50)  | 是  | 类别编码，如M、T、P、S         |
| category_name | VARCHAR(50)  | 是  | 类别名称                  |
| category_type | TINYINT      | 是  | 类型：1管理通道, 2专业通道, 3双通道 |
| description   | VARCHAR(200) | 否  | 类别描述                  |
| sort          | INT          | 是  | 排序                    |
| state         | TINYINT      | 是  | 状态：0禁用, 1启用           |

**标准职系定义:**

| 编码 | 名称  | 类型   | 说明                     |
|----|-----|------|------------------------|
| M  | 管理系 | 管理通道 | Manager，部门经理及以上        |
| T  | 技术系 | 专业通道 | Technology，工程师序列       |
| P  | 专业系 | 专业通道 | Professional，HR、财务、法务等 |
| S  | 销售系 | 专业通道 | Sales，销售、市场序列          |

#### 3.4.2 职级管理

**功能描述:**
管理各职系内的职级划分，定义晋升通道和薪资范围。

**核心字段:**

| 字段名          | 类型            | 必填 | 说明               |
|--------------|---------------|----|------------------|
| id           | BIGINT        | 是  | 主键               |
| category_id  | BIGINT        | 是  | 所属职系ID           |
| level_code   | VARCHAR(20)   | 是  | 职级编码，如T1、T2-1    |
| level_name   | VARCHAR(50)   | 是  | 职级名称             |
| level_seq    | INT           | 是  | 职级序列号，用于排序比较     |
| level_group  | VARCHAR(20)   | 否  | 职级分组：初级/中级/高级/专家 |
| min_salary   | DECIMAL(12,2) | 否  | 薪资范围下限           |
| max_salary   | DECIMAL(12,2) | 否  | 薪资范围上限           |
| stock_level  | TINYINT       | 否  | 股权激励等级           |
| requirements | TEXT          | 否  | 晋升要求（JSON）       |

**技术系职级示例:**

| 编码   | 名称    | 序列 | 分组   | 薪资范围     |
|------|-------|----|------|----------|
| T1   | 初级工程师 | 1  | 初级   | 8k-12k   |
| T2-1 | 工程师   | 2  | 中级   | 12k-18k  |
| T2-2 | 高级工程师 | 3  | 中级   | 18k-28k  |

#### 3.5.3 敏感操作审计

**功能描述:**
对高风险操作进行二次确认和审计，支持审批流程。

**审计类型:**

- 权限变更（角色分配、权限调整）
- 数据导出（批量导出用户数据）
- 批量操作（批量删除、批量禁用）
- 敏感数据访问（查看明文密码、导出薪资数据）

**审计流程:**

```
1. 用户发起敏感操作
       │
       ▼
2. 系统记录审计申请
   ├─ 标记操作日志为"待审计"
   └─ 生成审计记录
       │
       ▼
3. 通知审计人
       │
       ▼
4. 审计人审批
   ├─ 通过 → 执行操作
   └─ 驳回 → 拒绝操作
       │
       ▼
5. 记录审计结果
```
